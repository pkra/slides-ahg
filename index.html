Authoring with accessible web documents in mind.

The case for web documents.

  • Most accessible.
  • Most flexible.
  • Most sustainable.

Do one thing and do it well

  • Word
  • LaTeX
  • Markdown

It doesn't matter which as long as you do a good job with it.

Keep it simple, stupid.

  • Native content -- stay in your tool
  • Simple layout -- do less, get more
  • Convert early, convert often

Get from here to there

  1. Word => pandoc
  2. LaTeX => pandoc,latexml, tex4ht (don't), plastex, tralics
  3. Markdown => gazillion tools for web, also pandoc (e.g., to Word, LaTeX), also writage (Word plugin)

Checking results

  1. Learn to use a screenreader (NVDA, VO, Orca)
  2. Learn automated tools (aXe, ChromeLens, tota11y) and their limitations.
